3.0.0 - 2020-06-10
  • Complete rewrite in Kotlin Multiplatform. Currently supported plain JVM, Android, iOS, and JS platforms.
  • Parseltongue and rich message formatting has been removed in favor of only simple {} formatting tags.
  • Public API has been cleaned up and streamlined for the 80% use-case.
    • Log methods no longer have a dedicated Throwable parameter overload. Just add the throwable messages to the message formatting.
    • Removes multi-profile support. Instead, the need for multiple profiles can be accomplished simply be creating a new profile at that location.
    • Removes the tag stack, and just supports a single tag. Manage tag stacks yourself or use dedicated Profiles at the needed location.
